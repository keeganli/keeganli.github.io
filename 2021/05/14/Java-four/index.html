一、包装类及其用法

除了前面介绍的关于类、对象的基本语法外，本篇将继续介绍Java面向对象的特性。
Java为8个基本类型提供了对应的包装类，通过这些包装类可把8个基本类型的值包装成对象使用。同时JDK 1.5提供了自动装箱和自动拆箱功能，允许把基本类型值直接赋给对应的包装类引用变量，也允许把包装类对象直接赋给对应的基本类型变量。

基本数据类型和对应的包装类

byte     -------     Byte
short     -------     Short
int     -------     Integer
long     -------     Long
char     -------     Character
float     -------     Float
double     -------     Double
boolean -------     Boolean

自动装箱和自动拆箱

public class AutoBoxingUnboxing{
    public static void main(String []args){
        //直接将一个基本类型变量赋给Integer
    Integer inObj = 5;//(装箱) 基本数据-->包装类
    int it = inObj;//(拆箱) 包装类-->基本数据
    }
}

二、处理对象

1.toString方法

假定定义了一个Person类，在主函数时
有下面的语句
Person p = new Person("GaoMing");
System.out.println(p);
运行程序后，输出Person@15db9742
@后面的八位十六进制数字可能会改变
那这个输出是怎么产生的呢？
实际上，toString()方法是Object类里的一个实例方法，所有的Java类都是Object类的子类
因此，所有的Java对象都具有toString()方法。

因此，我们可以改写toString()方法，达到我们想要的输出方式

public String toString(){
    return "(想要的字符串格式)";
}

类成员

在Java类里只能包含
成员变量、方法、构造器、初始化块、内部类（包括接口、枚举）。

以static修饰的的成员/方法就是类成员/方法，属于类而不属于对象
这和C++相似，这里不再展开

三、抽象类

有些时候，我们对事物不能用具体的东西来描述，这个时候就应该把事物定义为抽象类。

抽象类的特点：

抽象类或者抽象方法必须用abstract修饰
抽象类中不一定有抽象方法，但是有抽象方法的类一定是抽象类
抽象类不能实例化
可以按照多态的方式实例化。
抽象类的子类
    要么是抽象类
    要么重写抽象类中的所有抽象方法

抽象类的成员特点：

成员变量
    可以是变量，也可以是常量
构造方法
    有。
    不能实例化，有构造方法有什么意义呢?
    用于子类访问父类数据的初始化。
成员方法
    可以是抽象的，也可以是非抽象的

abstract不能和哪些关键字共存?

final: 
    final修饰的方法不能被重写，而abstract修饰的方法要求必须被重写。
private：
    private修饰的方法不能被重写，而abstract修饰的方法要求必须被重写。
static：
    static修饰的方法可以通过类名调用，而abstract修饰的方法是没有方法体的，
    调用一个没有方法体的方法是没有任何意义的，所以不能共存。

抽象类应用举例

//抽象类Animal
abstract class Animal {
    int num = 10;
    final int num2 = 20;
    //抽象类构造不能实例化
    public Animal() {}
    //抽象成员方法
    public abstract void method();

    public abstract void function();
}
//继承应用
class Dog extends Animal  {
    public void show() {
        num = 100;
        System.out.println(num);
        //num2 = 200;
        System.out.println(num2);
    }
    //重写抽象方法
    public void method(){
    System.out.println("Dog");
    }
    public void function() {}
}

class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.show();
        d.method();
    }
}

程序输出：

100
20 
Dog

四、接口

有些时候，不是事物本身具备的功能，我们就考虑使用接口来扩展。
接口不提供任何实现，它体现的是规范和实现分离的设计哲学

接口的特点：

关键字interface

interface 接口名 {
    //各种定义，保证成一种规范
}
接口可实现多继承，即一个接口可有多个直接父接口，获得父接口的所有抽象方法、常量。

类实现接口用关键字implements 
    class 类名 implements 接口名 {}
    接口不能实例化
接口的子类
    抽象类
    或者重写接口中的所有方法

接口的成员特点

成员变量
    只能是常量（public static final）
成员方法
    只能是抽象方法（public abstract）

类与接口的关系

类与类
    继承关系，只能单继承，可以多层继承。
类与接口
    实现关系，可以单实现，也可以多实现。
    还可以在继承一个类的同时实现多个接口。
接口与接口
    继承关系，可以单继承，也可以多继承。

抽象类和接口

共性：
    都不能被实例化，位于继承树的顶端，用于被其他类实现和继承
    都可包含抽象方法，实现接口或继承抽象类的普通子类都必须实现这些抽象方法。
区别：
    抽象类：父抽象类，里面定义的是共性内容。
    接口：父接口，里面定义的是扩展内容。

接口应用样例

//定义接口类
interface Animal {
    //接口会自动指认静态常量分配static和fianl修饰符
    public static final int num = 10;
    public abstract void show();
}
//使用接口
class Dog implements Animal {
    public void show() {}
}

class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        //d.show();
        System.out.println(d.num);
    }
}

程序输出：

10
