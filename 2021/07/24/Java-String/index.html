Java's String

发表于 2021-07-24 更新于 2021-07-25 分类于 Java

创建字符串

public String(); 	\\创建String对象
public String(byte[] bytes); 	\\把字节数组转成字符串。
public String(byte[] bytes,int index,int length);	\\把字节数组中的一部分转成字符串
public String(char[] value);	\\把字符数组转成字符串
public String(char[] value,int index,int count);	\\把字符数组的一部分转成字符串
public String(String original);		\\把字符串转成字符串

在Java中，String实际上是引用类型，本质上也是一种class

字符串操作

接下来介绍常用的字符串处理函数

  • 字符串比较操作

    Java中字符串的比较要使用equals函数，而不是=

    //区分大小写比较
boolean equals(Object anObject)
String s1 = "GaoMing";
String s2 = "gaoming";
boolean flag = s1.equals(s2);
//不区分大小写比较
boolean equalsIgnoreCase(String anotherString)

  • 字符串长度

    public class Main {
    public static void main(String args[]) {
        String s1 = "GaoMing";
        int len = s1.length();  //访问器访问字符串长度
        System.out.println(len);
   }
}

  • 连接字符串

    1. string1.concat(string2);
2. string1 + string2;

  • 字符串转换

    要把任意基本类型或引用类型转换成字符串，可以使用静态方法valueOf()

    String.valueOf(123); "123"
String.valueOf(12.23); "12.23"
String.valueOf(true); "true"

  • 其他常用函数

    char charAt(int index)		//返回指定索引处的 char 值
    
int compareTo(String o)		//把这个字符串和另一个字符串比较。
    
boolean contentEquals(StringBuffer sb)		//当且仅当字符串与指定的StringBuffer有相同顺序的字符时候返回真。
    
static String copyValueOf(char[] data)		//返回指定数组中表示该字符序列的 String。

StringBuilder

Java编译器对String做了处理，可以使用+连接字符串，但是我们知道String本身不会变，不断创建新的临时字符串，然后扔掉，会造成内存浪费，效率比较低。

借此，Java提供了StringBuilder，它是一个可变对象，可以预分配缓冲区，可以提高效率。

StringBuilder的使用

StringBuilder sb = new StringBuilder(100);
for(int i = 0;i < 100;i++){
    sb.append(',');
    sb.append(i);
}
String s = sb.toString();

其实，对于普通的字符串+操作，并不需要我们将其改写成StringBuilder,因为Java编译器在编译时会自动把多个连续的+操作编码为StringConcatFactory的操作，在运行期间，StringConcatFactory会自动把字符串连接操作优化为数组复制或者StringBuilder操作。

小结

  • Java字符串String是不可变对象
  • 字符串的所有操作并不改变原字符串内容，而是返回新字符串
  • 善用String内置函数
  • StringBuilder是可变对象
  • StringBuffer相对于StringBuilder是线程安全版本的，效率较StringBuilder低
